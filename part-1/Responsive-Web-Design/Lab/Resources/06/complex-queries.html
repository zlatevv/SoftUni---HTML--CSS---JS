Complex Media Queries

Reference: MDN - Creating Complex Media Queries

Combining multiple types or features

The and keyword combines multiple media features. This example restricts styles to devices in landscape orientation and a minimum width of 30em. 

@media (min-width: 30em) and (orientation: landscape) {
    body::before {
        display: block;
        content: 'The viewport width is larger than 30em and the orientation is landscape.';
    }
}

Testing for multiple queries

A comma , separates multiple queries. Styles apply if any of the queries match. 

@media (min-height: 680px), screen and (orientation: portrait) {
    body::before {
        display: block;
        content: 'The viewport height is larger than 680px or the media type is screen and the orientation is portrait.';
    }
}

Inverting a query's meaning

The not keyword negates a single query. This example targets viewports not wider than 600px. 

@media not (min-width: 600px) {
    body::before {
        display: block;
        content: 'The viewport is NOT larger than 600px';
    }
}
